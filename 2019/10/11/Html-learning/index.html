html学习笔记

一、总述

html是一种浏览器“看得懂”的标记语言。当用浏览器打开html文档时，就可以呈现出作者想要呈现的样式。html文档的内容都由元素组成。这些元素一般包括“开始标签”，“属性”，“内容”，“结束标签”。其中大多元素内容部分可以嵌套元素。下面是一个html文档：

<!DOCTYPE html> （这是html5的文档类型元素）
<html> （这是html元素的开始标签）
<head>…</head> （这是html元素的内容，其中嵌套了一个head元素，head元素的内容被省略了）
<body>…</body> （同上）
</html> （html的结束标签）

其中head元素的内容大多不会被浏览器呈现给读者，是给浏览器看的，可以没有（但大多都有）。body元素的内容则大多会展现给读者并可以有不同的样式和功能。

二、元素与属性

在上一部分的html文档的基础上，剩下的所有标签都只应该出现在head元素和body元素的内容中。其中所有展示给读者的元素可以分为为块级元素（block level element）和内联元素（inline element）；其他的元素是只展示给浏览器看的辅助性标签。属性的作用是定义元素样式或增加一些功能。

1.块级元素

块级元素一般会在开始和结束换行，可以嵌套块级元素和内联元素，并且高度宽度可以通过属性来控制。

  • 以下的块级元素没有特殊样式：
    <h1>~<h6> ：标题。
    <p> ：段落。

    注意：它们都可以用style属性修饰。
    例 ：

    <h1 style=”text-align:center;color:red”>这是居中的红色的标题</h1>

这是居中的红色的标题

  • 以下的块级元素有特殊样式：
    <blockquote> ：长引用。
    <address> ：地址。
    <pre> ：预格式。

  • 列表元素 ：
    <ul> ：这是无序列表，有<li>子元素。
    <ol> ：这是有序列表，也有<li>子元素。
    <dl> ：这是自定义列表，有<dt>子元素，<dt>又有<dd>子元素。

  • <table> ：
    这是表格，有<caption>,<tr>,<th>,<td>等子元素，可以用border(类型)，frame（类型）,cellpadding(内容与边的距离),cellspacing(单元格间的距离)等属性修饰。需要合并单元格则用colspan（横着合并）或rowspan（竖着合并）属性修饰<th>或<td>元素。
    例：

    <table border=”1”>
    <caption>表格例子</caption>
    <tr>
    <th style=”text-align: left”>表头左对齐</th>
    <td bgcolor=”red”>红色的表</td>
    <td> <p>这里嵌套了一个段落</p></td>
    </tr>
    <tr>
    <th rowspan=”2”>竖向合并两个格子的表头</th>
    <td>&nbsp</td>
    <td></td>
    </tr>
    <td>这是用了实体的效果</td>
    <td>这是没有用的效果</td>
    </tr>
    </table>

2.内联元素

内联元素不会换行，只能嵌套内联元素，高度宽度不可控。

  • 以下的内联元素有特殊文字效果，能表现强烈视觉效果：
    <b> ：定义粗体文本。
    <big> ：定义大号字。
    <em> ：定义着重文字。
    <i> ：定义斜体字。
    <small> ：定义小号字。
    <strong> ：定义加重语气。
    <sub> ：定义下标字。
    <sup> ：定义上标字。
    <ins> ：定义插入字。
    <del> ：定义删除字。
    <bdo> ：通过dir属性表现文字方向。

  • 以下的内联元素能表现跟计算机相关的文字效果：
    <code> ：定义计算机代码，不保留空格和换行。
    <kbd ：>定义键盘输入。
    <samp> ：定义计算机输出。
    <tt> ：定义打字机代码。
    <var> ：定义数学变量。

  • 以下的内联元素能产生缩写的效果：
    <addr> ：缩写，可以title属性修饰，属性的内容即为完整内容。
    <acronym> ：首字母缩写，同上。

  • 以下的内联元素能产生引用的效果：
    <q> ：短引用。
    <cite> ：引用的出处 如著作标题。
    <dfn> ：定义项目。

  • <img/> ：
    定义一个图片，没有结束标签，通过使用scr属性来呈现图片（先参看URL内容（附一个书签）），有<map>,<area/>等子标签，可以用alt（替换文本），align(图片位置)，width(图像大小)等属性修饰。
    如何将图片加到库中？

    1. 在根目录source文件夹中新建一个image文件夹，将图片放在那里，用“/image/文件名”来使用
    2. 修改源文件的配置

  • <a> ：
    定义一个链接，通过href属性链接到一个地址，内容可以是文字或者嵌套图片等，可以用target（打开地址的方式），name（锚）等属性修饰。

    锚：简单来说就是书签，例如页面中有一个<a name=”tip”>…</a>的元素，则可以通过点击同一页面中的<a href=”#tip”>…</a>或不同页面中的<a href=”页面地址#tip“>…</a>直接跳转到这里。

    name属性与id属性的关系

3.<div>和<span>

这两个元素不呈现任何效果，但是能很方便的定义子元素的属性，当被id或class属性修饰时有更大的作用。

4.属性（全局属性）

有一些属性几乎可以修饰所有元素，叫全局属性；有一些属性只能修饰某种或某几种元素。以下是几个全局属性：

  • <id> ：给元素定义一个唯一的“名字“，可以用这个id链接更多属性，让定义属性更方便(加一个链接头文件的标签)，这个属性几乎可以修饰所有的元素，而name属性只能修饰<a>元素。
  • <class> ：给所有有相同class属性内容的元素链接同样的属性，与id差不多(加一个链接头文件的标签)。
  • <style> ：定义样式，比如文字效果，文本对齐方式等，例子见前面。

5.其他元素

这些元素有一定的功能效果：

  • <hr/> ：显示一个水平线或者说分割线。
  • <!– 内容 –> ：注释，不会显示给读者。
  • <br/> ：换行。

6.<head>元素的子元素

以下元素只能放在<head>元素中：

  • <title> ：定义html文档的标题，会被浏览器呈现在浏览器窗口的标题栏或状态栏上。
  • <base/> ：定义了html文档内所有链接的默认地址跟默认目标，如果没有则默认地址是网页的地址。需要用href和target属性修饰。
  • <link/> ：定义了与外部资源的关系，常用的只有链接样式表。例如<link rel=”stylesheet” type=”text/css” href=”theme.css” />，其中rel属性定义关系类型，这里的stylesheet即为样式表；type属性定义样式表用的语言；href属性定义链接的地址。
  • <style> ：定义了某些种类的元素的样式，可以用type属性修饰。例如<style type=”text/css”>h1 {color:red} p {color:blue}</style>，定义了文档中所有的<h1>的文字颜色都是红色，所有<p>的文字颜色都是蓝色。
    1. style与id，class属性连用可以方便的定义样式，内容的格式为：
      (元素类型).class属性的内容 (作用的某个子元素,另一个子元素) { 属性:内容 ; 属性:内容 }
      (元素类型)#id属性的内容 (作用的某个子元素,另一个子元素) { 属性:内容 ; 属性:内容 }
      • 样式会定义对应id或class属性修饰的元素
      • 如果有元素类型则其他类型的元素即使有对应id或class属性修饰也不会有样式
      • 如果有作用的子元素的话则样式会定义在子标签内
    2. style与div连用可以方便的布局页面，内容的格式与之前相同，主要通过float属性和width(限制块级元素的宽),heigh(限制元素的高)来实现，限宽之后的块级元素设置float属性为left，块级元素就会按顺序排列在一行(除非宽度超过了页面)。
  • <meta/> ：定义了html文档的元信息，让浏览器了解一些信息。如<meta charset=”gbk” />会告诉浏览器这篇文档是用gbk写的；<meta name=”keywords” content=”HTML, CSS, XML” />会告诉浏览器这篇文档的关键字是HTML，CSS，XML；<meta name=”description” content=”Free Web tutorials on HTML, CSS, XML” />会告诉浏览器这篇文档的描述是Free Web tutorials on HTML, CSS, XML。一些信息可以用于搜索引擎。

7.框架元素

之前的分类都建立在总述的例子上，但<body>标签可以被<frameset>标签代替，形成框架

  • <frameset>元素有<frame/>、<noframe>等子元素，也可以嵌套<frameset>元素实现复合框架。需要使用cols(横着分割)、rows(竖着分割)分割框架。<frame/>元素需要src属性修饰来定义页面。<noframe>元素用于表示如果浏览器不支持框架结构显示的文字内容，可以嵌套<body>标签。
  • <iframe> ：定义内联框架，有点像文本框，可以做<body>元素的子元素出现在文档的任意位置，需要src修饰来定义页面，可以被height（限高），width（限宽），frameborder（是否有边框）修饰。

三、路径

路径表示的是一个位置，这个位置可以有一个图片，一个html文档等等，它的不同表示方法有不同含义：

  • 绝对路径：一个完整的URL码
  • 相对路径：一个不完整是URL码
